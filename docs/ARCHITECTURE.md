```
│   PhysCausal:   元物理 → 物理 → 因果 → 感知 → 可解释的结论      │
```

**表层物理定律 (F=ma, PV=nRT) 告诉你「世界怎么运转」。**
**深层元物理规律告诉你「为什么世界只能这样运转」和「规律本身何时会变」。**

两者缺一不可：只知道 F=ma 无法解释为什么冰会融化（相变=对称性破缺）、为什么存在不可逆过程（熵增）、为什么观测会改变系统（测量坍缩）。

### 2.2 对称性 (Symmetry) 模块

```python
class Symmetry:
    """对称性 — 系统在某种变换下的不变性"""

    连续对称性:
      - 时间平移 T: t → t + δt
         → Noether: 能量守恒 dE/dt = 0
         → 因果约束: 方程中不能显含 t

      - 空间平移 X: x → x + δx
         → Noether: 动量守恒 dP/dt = 0
         → 因果约束: 方程中不能显含 x

      - 旋转 R: θ → θ + δθ
         → Noether: 角动量守恒 dL/dt = 0

    在因果模型中的应用:
      1. 检测变量的守恒律
      2. 将守恒律作为 DAG 的硬约束
      3. 验证反事实结果是否违反守恒

class SymmetryBreaking:
    """对称性破缺 — 新变量涌现的机制"""

    自发破缺:
      系统处于高对称态 → 微小扰动 → 坍缩到低对称态
      例: 水 → 冰 (连续平移对称 → 离散平移对称)

    因果含义:
      相变点 = 因果图的「结构断点」
      → 需要重新运行因果发现
      → 新变量可能涌现 (如: 晶体取向、磁化方向)

    显式破缺:
      外界施加的对称性破坏
      例: 加电场 → 空间各向同性破坏
      → DAG 中出现新的边
```

### 2.3 熵增 (Entropy) 模块

```
核心: 第二热力学定律 — ΔS ≥ 0

在因果推断中的应用:

  1. 因果方向判定:
     如果 X→Y 和 Y→X 在统计上等价 (Markov 等价类),
     选择「熵增方向」作为因果方向。

     原理: 因果过程产生熵。
           原因 → 结果 = 信息损耗 = 熵增。
           因此原因端通常熵更低 (更有序)。

     例: 
       送水 → 水变热    (熵增: 电→热, ΔS > 0)  ✓
       水变热 → 送水    (熵减: 热→电, ΔS < 0)  ✗

  2. 不可逆过程识别:
     哪些因果路径是可逆的? (保守力, 无熵增)
     哪些是不可逆的?   (耗散过程, 有熵增)

     例:
       碰撞 (弹性)     → ΔS=0  → 可逆
       摩擦生热        → ΔS>0  → 不可逆 (反事实中能量不守恒)

  3. 时间箭头验证:
     所有因果边必须沿时间正向 (原因→结果)。
     熵增方向 = 时间方向 = 因果方向。
```

**熵增 vs 熵减 — 智能体的核心张力**

元物理的熵增是铁律（不能违反）。但智能体的每一层都在做局域熵减：

```
层               熵流        原因
──────────────────────────────────────────
感知层            ↘ 熵减     像素(H高) → 语义变量(H低)
谱分解            ↘ 熵减     PCA 筛掉低方差维度
信息瓶颈          ↘ 熵减     minimize I(T;X) 丢掉冗余
因果发现          ↘ 熵减     稠密关联图 → 稀疏 DAG
SCM 拟合          ↘ 熵减     n 数据点 → n_params 参数
──────────────────────────────────────────
元物理约束        ↗ 熵增     宇宙收的「税」
                             Landauer 极限: 1 bit ≥ kT ln 2
```

智能体 = 局域熵减机器。用能量和信息为代价，把噪声提炼为结构。
净效果：ΔS_宇宙 > 0 永远成立——智能体减少的熵 < 环境增加的熵。

这就是 `information/` 横切层存在的意义：
- `meta_physics/entropy.py` — 熵增是铁律（不能违反的约束）
- `information/bottleneck.py` — 熵减是目标（智能体存在的意义）
- `information/shannon.py` — 两者的共同数学语言

**压缩即理解 — 从 Shannon 到 Solomonoff**

压缩不只是「省空间」。数学上，压缩能力等价于理解能力：

```
Kolmogorov 复杂度 K(x):
  生成 x 的最短程序的长度。

Solomonoff 归纳:
  在所有能解释数据的程序中，最短的那个是最好的预测器。
  短程序 = 抓住了规律。长程序 = 记住了每个细节（过拟合）。

Occam 剃刀的数学形式:
  P(h|D) ∝ P(D|h) · 2^{-K(h)}
  假设 h 的先验概率与其描述长度成反比。
  越短的假说越可能为真。
```

PhysCausal 的每一层都是压缩操作：

```
感知压缩:   1000×1000×3 像素 → 20 维语义变量
            丢掉了每个像素的精确颜色
            保留了「那里有个红色的球」

谱压缩:     50 维特征 → 8 维主成分
            丢掉了低方差噪声方向
            保留了数据变化最大的方向

因果压缩:   稠密关联图 → 稀疏 DAG
            丢掉了虚假相关
            保留了真正的因果边

SCM 压缩:   500 个样本点 → 3 个参数 (β₀, β₁, σ)
            丢掉了个体波动
            保留了结构性规律
```

每一层做的是同一件事：**舍弃不可压缩的噪声，提取可压缩的规律。** 整个 PhysCausal 可以概括为一句话：

> 从原始数据中提取最短的因果程序。

### 2.4 信息边界模块

```
核心: 量子力学中，观测使系统从叠加态 |ψ⟩ 坍缩到本征态 |φ_i⟩

经典类比 (对因果推断至关重要):

  叠加态 = 反事实可能性的集合
  坍缩   = do() 干预 — 从所有可能性中选择一个

  "在测量之前，粒子同时在这里和那里"
  "在观察之前，张三可能上大学也可能不上"
  
  → 这是同一个逻辑结构!

Pearl 三步反事实 与 量子测量的对应:

  Abduction  = 制备态   (从观测反推 U，如从测量反推 |ψ⟩)
  Action     = 测量     (施加 do()，如选择测量基)
  Prediction = 坍缩     (得到确定结果，如本征值)

关键洞察:
  观测不是被动的记录，而是主动的干预。
  每次测量 = 一次 do() 操作。
  不同的测量基 = 不同的干预变量。
```

### 2.7 元物理的完备性：五条第一性原理

当前实现了四条（对称性、熵增、测量、特征谱）。完整的元物理地基应包含五条：

```
Tier 0 — 生成性原理 (为什么有规律)
────────────────────────────────────
  ① 最小作用量 δS = 0
     所有动力学方程从此导出。
     在因果推断中: 最优因果路径是作用量最小的路径。
     如果一条因果链违反 δS=0，它在物理上不可能发生。

  ② 对称性 → 守恒 (Noether)
     为什么守恒律存在。
     在因果推断中: 守恒量是反事实推理的硬约束。
     反事实结果必须满足 ΔEnergy=0, ΔMomentum=0。

Tier 1 — 过程约束 (规律怎么运作)
────────────────────────────────────
  ③ 熵增 ΔS ≥ 0
     为什么时间有方向。
     在因果推断中: 因果箭头必须沿熵增方向。
     用于打破 Markov 等价类的对称性。

  ④ 局域因果 (Locality)
     为什么因果必须在光锥内，不能超距作用。
     在因果推断中: DAG 的边必须满足时间序。
     任意两个无因果联系的变量必须 d-separated。

Tier 2 — 信息边界 (我们能知道什么)
────────────────────────────────────
  ⑤ 信息边界
     为什么获取信息必然改变系统。
     在不同尺度有不同名字，同一种数学:
       量子: 波函数坍缩 — |ψ⟩ → |φ_k⟩
       经典: 条件化 — P(Y) → P(Y|X=x)
       因果: do() 干预 — E[Y] → E[Y|do(x)]

     在因果推断中: 每次数据收集 = 从可能世界集合中选出一个。
     反事实 = 在另一个投影子空间中做计算。
```

### 4.1 贝叶斯推断与因果推断的关系

贝叶斯推断和因果推断不在同一个维度上——不是替代关系，是正交互补。

```
                无因果                      有因果
─────────────────────────────────────────────────────
频率学派    P(Y|X) = 回归系数          do-calculus + OLS
            「X和Y相关」              「do(X)对Y的效应」

贝叶斯      P(θ|D) ∝ P(D|θ)P(θ)      P(θ, G|D) ∝ P(D|G,θ)P(θ|G)P(G)
            「我对θ的信念」            「我对因果结构和参数的信念」
```

PhysCausal 站在右下角——**贝叶斯因果推断**。

**分工：**

```
causal 回答:     「X 对 Y 的因果效应是多少？」
bayesian 回答:   「你对这个答案有多大把握？」
                 「如果数据增加 10 倍，答案会变多少？」
                 「要不要做实验？做哪个？」
```

**包裹关系（不替代）：**

```
bayesian/          ← 不确定性壳
  │ 包裹
causal/            ← 因果引擎 (不变)
  │
  PC → DAG          原来: 一个 DAG
  PC → P(G|D)       现在: DAG 的分布

  OLS → ATE          原来: 一个 ATE
  OLS + prior       现在: ATE 的后验
     → P(ATE|G,D)
```

bayesian/ 不做因果推断——它不替代 do-calculus、不替代 SCM。它**包裹**因果引擎，输出概率而非点估计。

**对应 P1 缺口：**

```
GAP_ANALYSIS.md P1:
  ✅ 不确定性量化 — Bayesian SCM (structural.py + parameter.py)
  ✅ 主动实验设计 — VOI 驱动 (active_learning.py)
```

## 十、从元物理到反事实的完整数据流

```
输入: 摄像机拍到球碰撞

┌─ 感知层 ─────────────────────────────────────────────┐
│ 像素 → [球A: {x,y,vx,vy,m}, 球B: {x,y,vx,vy,m}]     │
└────────────────────┬─────────────────────────────────┘
                     ▼
┌─ 元物理层 ───────────────────────────────────────────┐
│                                                       │
│ 对称性检测:                                            │
│   时间平移不变? ✓ → 能量守恒应成立                      │
│   空间平移不变? ✓ → 动量守恒应成立                      │
│                                                       │
│ 熵增检测:                                              │
│   碰撞前后: 宏观动能 → 微观热运动?                      │
│   弹性碰撞: ΔS≈0 → 可逆过程                            │
│                                                       │
│ 测量坍缩:                                              │
│   摄像头采样 = do(观测) → 系统从连续态坍缩到离散帧      │
│                                                       │
│ 输出约束:                                              │
│   - 动量守恒: m1v1+m2v2 = m1v1'+m2v2'                 │
│   - 能量守恒: ½m1v1²+½m2v2² = ½m1v1'²+½m2v2'²       │
│   - 因果方向: 碰撞发生 → 速度改变 (熵增方向)            │
└────────────────────┬─────────────────────────────────┘
                     ▼
┌─ 物理层 ─────────────────────────────────────────────┐
│ F=ma + 动量守恒 → 碰撞后的速度由碰撞前状态唯一确定      │
│ → SCM 中速度的方程被替换 (不靠统计拟合)                │
└────────────────────┬─────────────────────────────────┘
                     ▼
┌─ 因果层 ─────────────────────────────────────────────┐
│ DAG: v_A, v_B, m_A, m_B → collision → v'_A, v'_B    │
│ SCM: 物理方程驱动                                     │
│ do(v_A = 2×): → v'_B 减少? → 反事实计算               │
└────────────────────┬─────────────────────────────────┘
                     ▼
┌─ 推理引擎 ───────────────────────────────────────────┐
│ 反事实: "如果 A 速度减半, B 还会进洞吗?"               │
│   → 修改 SCM: v_A = v_A/2                            │
│   → 守恒验证: 动量+能量仍然守恒 ✓                      │
│   → 答案: B 速度不足, 不会进洞                         │
│                                                       │
│ 归因: "B 进洞的关键因素?"                              │
│   → v_A 贡献 62%, 碰撞角度 贡献 38%                    │
│                                                       │
│ 熵增验证:                                              │
│   整个过程 ΔS ≥ 0 满足                                 │
│   → 反事实在物理上可行                                 │
└───────────────────────────────────────────────────────┘
```
