### 2.1 主动因果实验设计 (Active Causal Discovery) ✅

支持 7 个物理仿真环境, physics_prior 物理优先跳过 PC, <1s per env。

```

核心能力:
  - 给定部分因果图，计算每条边的信息价值 (VOI)
  - 「做实验 X 能最大程度缩小剩余 DAG 的不确定性」
  - 实验设计预算约束下的最优干预序列

理论基础:
  - Causal Bandit (每一步选择一个干预)
  - Bayesian Optimal Experimental Design
  - Mutual Information Maximization: argmax_a I(G; D ∪ {do(a)})

优先级: P0 (致命) — 没有这个，永远是被动的分析工具
```

## 2.5 涌现 vs 可建模 — 抽象能力的分工

P0 缺口中的「层次化抽象」和「组合泛化」涉及的不仅仅是工程问题。
它们触及一个更根本的问题：**什么必须显式建模，什么只能靠规模涌现？**

当前 AI 的两个极端:

```
纯涌现路线 (GPT):            纯建模路线 (符号AI):
  一切从规模中自发产生          一切靠规则显式编码
  优点: 能处理模糊语义          优点: 精确、可解释
  缺点: 不可控、不可解释        缺点: 无法处理开放世界
```

PhysCausal 需要的不是选边，而是**分工**:

```
可以显式建模 (放在架构里)       只能从规模涌现 (需要数据+计算)
────────────────────────────────────────────────────────
组合的语法: 模块怎么拼          组合的语义: 拼出来是什么意思
  范畴论 / 接口定义              GPT 做常识推理

抽象的方向: 丢什么保留什么       抽象的直觉: 丢多少恰到好处
  信息瓶颈 / 重整化群            需要经验调 β 参数

类比的算子: 结构相似性度量       类比的灵感: 把球和月亮联系起来
  图核 / 结构同构检测            远距离联想

因果层次: 变量粗粒化的规则       因果层次: 新变量叫什么名字
  τ-互模拟 / 因果一致性          语义标注

关联记忆: 注意力机制             关联广度: 跨域映射
  Transformer / GNN             创意类比
```

对 PhysCausal P0 缺口的指导:

```
组合泛化 (§2.3):
  可建模 ─ 模块接口 (端口/类型签名)
           组合规则 (不交并/共享变量对接)
           类型检查 (因果变量兼容性)
  
  需涌现 ─ 「推车上的摆锤」和「钟摆」是同一类
           跨域类比识别 → 需要经验或搜索

层次化抽象 (§2.2):
  可建模 ─ 重整化群 (什么时候粗粒化)
           因果互模拟 (粗粒化后保持什么)
           信息瓶颈 (丢多少)
  
  需涌现 ─ 自动发现「温度」是分子的宏观序参量
           创造性抽象 → 需要搜索+验证

主动实验 (§2.1):
  可建模 ─ VOI 计算 (information gain)
           实验候选排序
           贝叶斯更新规则
  
  需涌现 ─ 「这个实验即使信息增益不高,
            但可能打开全新的因果假设空间」
           科学直觉 → 需要好奇心 + 元认知
```

核心结论:

> 抽象的语法可以设计，抽象的语义需要涌现。
> 联想的结构可以编码 (图核、类比算子)，联想的灵感需要规模。
> PhysCausal 的方向: 把能建模的语法层做扎实，给涌现留好接口。

### 2.6 随机性玄学 — 涌现的机制

涌现不是凭空发生的。它的机制可能是：**随机生成 + 确定性过滤 = 创新**。

进化论就是这么干的：随机变异 + 自然选择 = 新物种。创新在本质上可能就是认知的进化。

但关键不是「要不要随机」——是**把什么东西随机起来**。

**随机性的分层设计：**

```
层级              随机注入              为什么随机            过滤机制
──────────────────────────────────────────────────────────────────
表征层            随机投影              丢掉的维度可能有新视角  方差保留率
(感知/谱)         JL 变换              偶尔的「错误压缩」      explained variance
                                        发现隐藏结构

结构层            随机图变异            条件独立性检验的盲区    物理约束(五原则)
(因果发现)        加/删/反转边          PC只能发现等价类       BIC 得分
                                        随机跳出去可能发现      守恒律验证
                                        统计上不确定的真结构    局域因果验证

参数层            MCMC 采样后验         不确定性本身就是信号    后验概率
(SCM)            不选MAP，采样整个分布  低概率区域可能隐藏        marginal likelihood
                                        更好的解释

假说层            加权随机采样          均匀随机=99.9%垃圾      物理约束过滤
(创新生成)        按类型签名相容性       按结构相容性加权后       BIC vs baseline
                  采样候选组合            采样 → 大部分落在       人类反馈
                  然后坍缩到最优          「有意义的邻域」        保留入库
                                        → 量子测量式的随机

元认知层          随机目标生成           「这个方向看起来        长期回报验证
(好奇心)          「为什么不试试X?」    没意义，但万一呢？」      惊喜度量
                                        探索 vs 利用的权衡      information gain
```

**什么不该随机：**

```
不该随机          原因                      替代方案
─────────────────────────────────────────────────────────
物理定律          宇宙已经给了确定性答案     作为硬约束，不是探索目标
守恒律            违反直接丢弃               作为过滤标准
因果方向          熵增方向/时间序是铁律      用第二定律/局域因果定向
安全约束          do(危险干预) 不能随机试    仿真环境 + 软约束
已验证的模块      已经确定有效的因果结构      保留但允许组合
```

**核心原则：**

```
随机性注入的四个条件:
  1. 有明确的过滤标准 — 随机完能判断好坏
  2. 过滤标准可靠       — 物理定律 >> BIC >> 人类反馈
  3. 失败成本低         — 仿真 > 真实世界实验
  4. 探索/利用可切换    — 不确定时探索，确定时利用

随机 ≠ 均匀随机:
  好的随机是「在结构相容的空间里加权采样」。
  不是从所有可能中均匀抽 —— 而是像量子测量:
    可能性空间 Σ c_i|φ_i⟩ → 按 |c_i|² 加权 → 坍缩到一个本征态
  模块组合同理:
    不是任意两个模块随机拼
    而是按类型签名相容性加权采样候选对
    大部分落在「有意义的邻域」里
    再通过过滤坍缩到最优解

  这条原则不是凭空来的 — 它正是 §2.5.5 信息边界的核心:
    坍缩 = 可能性空间 → 向最可能方向投影
    量子测量 / 经典条件化 / 因果 do() / 模块组合采样
    四者是同一种数学操作在不同空间中的投影
```

**对 P0 缺口的指导：**

```
主动实验 — 确定性 VOI 走 80% 的路
           随机实验 (偶尔选低VOI但新奇的干预) 负责突破

层次抽象 — 确定性重整化群给出候选粗粒化
           随机粗粒化搜索偶尔发现更好的宏观变量

组合泛化 — 确定性接口对接
           随机组合 + 过滤 → 发现新的有效组合
           模块库越大，随机组合涌现新结构的概率越高
```

**一句总结：**

> 确定性架构决定下限，随机性决定上限。
> 我们当前只搭了下限。把随机性注入对的地方，
> 用物理和因果做过滤，才是从「可靠的工具」
> 到「有创造力的智能体」的跨越。
